Este middleware tem como objetivo garantir a confidencialidade, integridade, autorização e autencidade dos dados na comunicação entre sistemas distribuídos utilizando as especificações de web services security. Sendo que, através do serviço de segurança disponibilizado, qualquer aplicação já desenvolvida pode incorporar funções de criptografia, autenticação, assinatura digital e permissoes de acesso aos dados baseados em regras sem a necessidade de maiores implementações, uma vez que a implementação é disponibilizada pelos serviços de segurança.

Para efetuar a comunicação o administrador deve cadastrar os usuários criar as regras de comunicação para os respectivos sistemas na qual o usuário irá se comunicar e suas permissões de acesso as informações. Com base nestas regras a aplicação enviará os dados para o middleware. Este será o responsável por implementar as questões de segurança e enviar os dados através de web services para outra aplicação na qual receberá os dados através do middleware, que a entregará novamente a aplicação com base nas regras cadastradas, garantindo a segurança dos dados e a transparência na questão da implementação de todas as funcionalidades para suportar o WS-Security e demais camadas de rede.
